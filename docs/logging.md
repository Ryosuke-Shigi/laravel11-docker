# Logging

このドキュメントは、ログの分類、残してよい情報、残さない情報をまとめます。

ログは調査と運用判断を助けるためのものです。秘密情報、外部APIの生レスポンス、不要な個人情報を保存する場所ではありません。

## 基本方針

- ログは Job / Action / Service など、処理単位が分かる場所で扱う
- Repository では原則ログを出さない
- Repository は DB 操作や外部データ取得の境界に限定し、ログ判断は呼び出し側へ戻す
- DTO / ListDTO にログ出力責務を持たせない
- Controller は入口として必要最小限の失敗検知に留め、業務判断ログを抱えない
- APIキー、トークン、DBパスワード、AWSキー、SSH情報、`.env` の実値は出力しない
- 外部APIレスポンスを丸ごと保存しない

## ログ分類

### sync

API同期、バッチ、Job系の開始、終了、件数、スキップ理由を記録します。

残す項目:

- 処理名
- 対象種別
- 対象IDまたは検索条件の識別子
- 開始時刻、終了時刻、処理時間
- 取得件数、保存件数、更新件数、スキップ件数
- 失敗件数、削除件数
- Job ID、Command名、Action名
- 同期の成功または失敗

残さない項目:

- 外部APIの生レスポンス全文
- APIキー、アクセストークン、署名値
- `.env` の実値
- 不要な個人情報
- 保存判断に不要なリクエスト全文

### external_api

YouTube API、APIs.guru など、外部API呼び出しの成否、レート制限、リトライ判断に必要な情報を記録します。

残す項目:

- 外部サービス名
- endpoint名または操作名
- 呼び出し対象
- HTTP method
- HTTP status
- 失敗理由
- リクエストID、レスポンスIDなどの調査用ID
- レート制限や quota に関する安全なメタ情報
- リトライ有無、リトライ回数、待機時間、タイムアウト有無

残さない項目:

- APIキー、Authorization header、署名付きURL
- リクエスト body / レスポンス body の全文
- 外部APIレスポンス全文
- 巨大payload
- 個人情報
- Cookie、session、secret
- 本番接続文字列

### error

例外、失敗、復旧不能な状態を記録します。

残す項目:

- エラー種別
- 例外クラス
- 安全なエラーメッセージ
- 処理名
- 発生箇所
- 対象ID
- stack trace
- 再実行可能かどうか

残さない項目:

- 秘密情報を含む可能性がある request / response 全文
- DBパスワードや接続文字列
- 外部APIキー
- ユーザー入力の無加工全文

### operation

運用上の操作、手動実行、deploy確認、maintenance を記録します。

残す項目:

- 手動同期、保存、削除、設定変更などの操作名
- 実行者を識別できる安全な情報
- 対象環境
- 対象コマンド名
- 開始時刻、終了時刻
- 結果
- 確認した範囲

残さない項目:

- 本番 `.env` の内容
- SSH秘密鍵、接続情報、パスワード
- APIキーやクラウド認証情報
- 管理画面の認証情報

## レイヤー別の扱い

- Job: 非同期処理の開始、終了、失敗、リトライを記録する
- Action: ユースケース単位の結果、件数、分岐理由を記録する
- Service: 業務判断の結果を必要最小限で記録する
- Repository: 原則ログを出さず、結果や例外を呼び出し側へ返す
- Responder: レスポンス整形のためにログを出さない
- Component: 画面表示責務に限定し、サーバーログ方針を持たせない

## ログの責務境界

- Repositoryでは原則ログを出さない
- RepositoryはDB操作に集中する
- Job / Action / Service 側で、処理単位が分かるログを出す
- 秘密情報・APIキー・.env実値・巨大payload・個人情報はログに残さない

## 判断に迷う場合

ログに残してよいか迷う情報は残しません。

調査に必要な場合は、値そのものではなく ID、件数、状態、時刻、分類名などの安全なメタ情報に置き換えます。
